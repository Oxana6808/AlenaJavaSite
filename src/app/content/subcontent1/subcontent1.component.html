Command Line. Методы ввода и вывода.

1. Консоль(command line) и её виды

Консоль — это программа, которая позволяет показывать текстовые сообщения и принимать текст от пользователя. С помощью консоли можно управлять программами и давать им команды, например, вводить данные или получать ответы.

Виды консолей:

  • Командная строка (Command Line) — это тип консоли, с которым мы работаем в операционных системах, таких как Windows, Linux или macOS. Это текстовый интерфейс, где вы запускаете программы и видите результаты их работы.
  • Если у вас Windows, то консоль вы можете открыть командой Win+r и введя cmd.

2. Методы ввода и вывода

Для вывода данных мы используем несколько методов:

  • print() — выводит данные в консоль без перехода на новую строку.
  • println() — выводит данные с переводом курсора на новую строку.
  • printf() — выводит данные в формате с использованием специальных шаблонов (например, для чисел и строк).

Пример использования методов:

// Пример использования методов print, println и printf
public class PrintMethodsExample {
    public static void main(String[] args) {
        // Пример метода print()
        System.out.print("Hello, ");
        System.out.print("world!");

        // Пример метода println()
        System.out.println(" Это пример println.");
        System.out.println("Новая строка начинается здесь.");

        // Пример метода printf()
        int age = 25;
        System.out.printf("Ваш возраст: %d лет\n", age);
    }
}

В результате выполнения этой программы на консоль будет выведено:

Hello, world! Это пример println.
Новая строка начинается здесь.
Ваш возраст: 25 лет

Для ввода данных используем класс Scanner. С его помощью можно получать строки, числа и другие типы данных от пользователя.

Scanner scanner = new Scanner(System.in);
System.out.print("Введите ваше имя: ");
String name = scanner.nextLine();
Введите ваше имя: Алёна
Привет, Алёна!

3. Пять этапов компиляции программы на java.

Написание исходного кода (HelloWorld.java): Программист пишет исходный код программы на языке Java и сохраняет его в файле с расширением .java. Например, это файл HelloWorld.java. 

// HelloWorld.java, название файла совдает с именем класса
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, World!");
    }
}

Компиляция исходного кода (Java Compiler): Компилятор Java (javac) компилирует исходный код из файла HelloWorld.java в байт-код. Байт-код — это промежуточный машинный код, который понимает Java Virtual Machine (JVM). Компилятор проверяет код на синтаксические ошибки и преобразует его в файл с расширением .class. Этот этап переводит код с высокоуровневого языка в машинный язык, который не зависит от конкретной платформы.

javac HelloWorld.java

Если компиляция прошла успешно, создается файл HelloWorld.class без вывода ошибок.

Создание байт-кода (HelloWorld.class): После успешной компиляции создается файл с расширением .class, который содержит байт-код программы. Например, это файл HelloWorld.class. Байт-код представляет собой набор инструкций, которые могут быть выполнены на любой платформе, где установлена JVM. Этот этап также включает проверку корректности байт-кода на предмет ошибок или потенциальных угроз безопасности.

javap HelloWorld.class

Пример вывода байт-кода:

Compiled from "HelloWorld.java"
public class HelloWorld {
  public HelloWorld();
  public static void main(java.lang.String[]);
}

Выполнение байт-кода на JVM (Java Virtual Machine): Виртуальная машина Java (JVM) загружает и выполняет байт-код из файла .class. JVM интерпретирует байт-код или компилирует его в машинный код для конкретного устройства (с помощью JIT-компиляции — Just-In-Time). JVM является кросс-платформенной и может работать на любой операционной системе, для которой существует реализация JVM.

java HelloWorld

Запуск программы (Program Execution): после того как JVM интерпретирует или компилирует байт-код в машинный код, программа запускается на конкретном компьютере. Пользователь видит результаты работы программы, например вывод в консоль или интерфейс графического приложения. На этом этапе программа выполняет логику, которая была заложена разработчиком в исходный код.

Вывод программы:

Hello, World!
